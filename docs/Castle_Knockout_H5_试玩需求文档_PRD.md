# Castle Knockout H5 试玩需求文档（H5skill 重构版）

## 一、项目概述

- 玩法品类：颜色匹配 + 蓄力投射 + 3D 物理拆除。
- 画面与视角：竖屏 9:16，玩家位于城堡正前方；地面纵深为 Z 轴，画面横向不参与瞄准。
- 试玩时长：15–25 秒。
- 核心卖点：按住决定射程、同色连锁爆破、砖块向屏幕深处飞散、结构失去支撑后二次解体。
- 转化目标：完成或用尽炮弹后展示 End Card，DOWNLOAD 跳转 TapTap。
- 参考依据：用户提供的原作商店页、宣传图、明确的 Z 轴操作反馈，以及公开商店描述中的颜色匹配与物理连锁规则。

## 二、核心玩法定义

### 一句话玩法

按住屏幕让落点沿 Z 轴向城堡深处移动，在合适射程松开，将当前颜色炮弹投进对应深度的同色砖群，触发连锁清除和结构解体。

### 核心循环

1. HUD 显示当前炮弹颜色、下两发颜色和剩余炮弹。
2. 按住任意游戏区域开始蓄力；落点只沿中心纵深线移动。
3. 松开发射；炮弹从画面外前景沿抛物线飞向屏幕深处，并随透视缩小。
4. 命中正确深度的同色核心砖群，触发同色连锁。
5. 失去支撑的周边砖延迟解体，砖块向消失点飞远并旋转缩小。
6. 完成 5 个核心目标后城堡整体崩解，进入 End Card。

### 关键规则

- 禁止 XY 拖拽瞄准；`pointermove` 不改变横向或纵向屏幕坐标。
- 蓄力时间 0–1.25 秒映射到 Z 轴射程 0–100%。
- 三个主要深度区：前庭约 20%、中墙约 52%、后塔约 84%。
- 当前炮弹只能清除指定深度内的同色核心砖群。
- 命中容错：目标深度前后各约 18%；首发再增加轻度磁吸，保证 5 秒内理解。
- 错误深度也会击飞少量普通砖并给出 `NEAR MISS`，不出现硬失败打断。
- 固定目标序列：蓝前庭 → 红中墙 → 金后塔 → 红中墙 → 蓝前庭。
- 共 6 发，5 个核心目标，保留 1 发容错。

## 三、试玩阶段

### 阶段 0：Hook（0–2 秒）

- 画面：大型多砖城堡占据画面主体；第一组蓝色核心砖脉冲高亮；前景蓝色炮弹半露在屏幕底部。
- 玩家目标：理解“当前蓝球要打蓝砖”。
- 自动兜底：2 秒无操作时显示按住手势和纵深虚线；目标圈自动从近到远演示一次。
- 反馈：城堡轻微呼吸、核心砖闪光、当前炮弹弹性浮动。
- 埋点：`CHALLENGE_STARTED`。

### 阶段 1：Tutorial（首次按住与松开）

- 操作：按住任意位置；落点沿中心 Z 轴移动。松开发射。
- 正确判定：释放射程进入当前核心目标容错区。
- 错误处理：偏差较小时吸附；偏差较大时产生近失爆破并保留当前颜色目标。
- 视觉反馈：Z 轴深度轨迹、动态落点环、蓄力条、目标核心同色连线。
- 音效反馈：蓄力升调、满蓄力提示、松手发射音。
- 埋点：`FIRST_INTERACTION`。

### 阶段 2：Core Play（约 3–16 秒）

- 每次成功命中：同色砖连续闪爆，结构支撑计算，失去连接的砖延迟飞散。
- 每发必须改变城堡轮廓并提升目标进度。
- 当前炮弹与 NEXT 队列更新；新的目标核心立即高亮。
- 进度节点：完成第 2/3/4 个目标时分别触发 25/50/75 进度事件。
- 埋点：`CHALLENGE_PASS_25`、`CHALLENGE_PASS_50`、`CHALLENGE_PASS_75`。

### 阶段 3：Climax / Result（约 16–20 秒）

- 第 5 个核心目标被清除后，剩余城堡砖分批向屏幕深处爆散。
- 屏幕震动、冲击波、粒子、慢动作和 `CASTLE CLEARED!` 大字同时出现。
- 1.2 秒后进入 End Card。
- 埋点：`CHALLENGE_SOLVED`。

### 阶段 4：End Card

- 成功：城堡缺口与皇冠奖章为主视觉，标题 `CASTLE CLEARED!`。
- 炮弹耗尽：弱失败文案 `SO CLOSE!`，仍展示成果与继续挑战。
- 主按钮：`DOWNLOAD`，链接到 `https://www.taptap.cn/moment/791302421241397589`。
- 次按钮：`PLAY AGAIN`。
- 埋点：`CLICK_CTA`。

## 四、状态机

```text
INTRO -> READY -> CHARGING -> FLYING -> IMPACT -> CASCADE -> READY
                                              -> CLIMAX -> RESULT -> END_CARD
READY / CASCADE + shots=0 -> RESULT -> END_CARD
END_CARD -> PLAY_AGAIN -> INTRO
```

## 五、物理与投影规范

- 炮弹使用归一化飞行进度，不依赖屏幕 XY 拖拽速度。
- 地面路径从屏幕底部向消失点插值，垂直高度使用 `sin(πt)` 抛物线。
- 炮弹半径随深度递减，阴影沿地面 Z 路径移动。
- 砖块拥有 `vx / vy / vz` 与三轴旋转速度；直接受击以正 `vz` 为主。
- 正 `vz` 通过透视投影向消失点收缩，表现为向屏幕内部飞远。
- 支撑判断按同一深度层的上下左右连接计算；不再简单删除命中点周围固定数量砖。
- UI/HUD 不参与镜头抖动；只震动游戏世界层。

## 六、防流失机制

- Idle Guidance：2 秒无操作自动演示 Z 轴落点移动。
- Input Forgiveness：首次命中扩大容错；接近目标自动吸附。
- Assisted Shot：连续近失后下一次进入更强磁吸范围。
- Fail-safe：最后一发对目标深度提供强辅助，避免无反馈结束。
- Audio Safe：首次交互后创建声音；静音按钮始终可见。
- Orientation：竖屏优先，桌面保持完整竖屏画布，不横向拉伸。

## 七、资源与反馈清单

- 背景：卡通城堡山谷、旗帜、城墙、暖金与天蓝配色。
- 砖块：石灰、蓝、红、金四色；正面、顶面、侧面与裂纹状态。
- 炮弹：蓝/红/金三色能量球，带皇冠标记。
- 动效：目标脉冲、蓄力纵深线、抛物线尾迹、同色连锁、冲击波、飞砖、粉尘、火花、最终爆破。
- 音效：蓄力升调、发射、命中低频、同色连锁高频、结构解体、胜利三音阶。
- 触觉：发射短振、命中组合振、满蓄力提示振。

## 九、验收标准

- 首屏 1 秒内可见大型城堡、当前炮弹和同色目标。
- 3 秒内能理解颜色匹配；5 秒内能完成首次有效发射。
- 手指移动不影响 XY 瞄准，只有按住时长影响 Z 轴射程。
- 炮弹从屏幕外前景进入，轨迹为抛物线并随深度缩小。
- 每次成功命中至少产生同色清除、支撑解体、震屏、音效、粒子与进度变化。
- 直接受击砖以向屏幕内部飞远为主要运动，不是整墙向下倾倒。
- 430×760 与常见 9:16 移动端无横向溢出，HUD 不被安全区裁切。
- 完成或炮弹耗尽后出现 End Card；DOWNLOAD 指向指定 TapTap 链接。
- 静态导出可在 GitHub Pages 子路径运行，测试与浏览器控制台无错误。
